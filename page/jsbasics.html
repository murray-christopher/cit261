Objects

Objects can be thought of as an a real life object and its description. Such as a ball that has a shape, color, texture, and material. A Balls object would look like this:

var ball = {
 shape: "Sphere",
 color: "red",
 texture: "smooth",
 material: "rubber"
};

On the left is the property and the right is the value. With this you can gather input from the user about a anything and store it as an object that I can then use to customise the user experience. I cover objects more in depth here

Arrays

Arrays are special types of objects. The main difference is that arrays do not need properties they are just a list of values. However they can store a list of object type and mixed datatypes just like objects, in which you can access using the dot operator or brackets to access elements. The next question you might ask is what one do you use where and why.

It comes down to good developer practices because you could get away with doing either one, however if your code calls for an array where your accessing positions in the object and looping through the values an array would be the better practice and make more sense. If your keeping track of a lot or objects that relate then an array that would be much better. More simply if its a list of related values I would use and array. If your trying to provide a description of something in code then use an object.
